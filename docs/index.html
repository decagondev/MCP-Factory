1 Minimal Server

Three Lines to a Running Server

Create the absolute minimum MCP server that runs and waits for client connections.

nasa_server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("nasa-apod")


def main():
    """Run the MCP server."""
    mcp.run(transport="stdio")


if __name__ == "__main__":
    main()
FastMCP("nasa-apod") creates a named server instance
transport="stdio" — local communication mode
2 API Helper

Async HTTP Requests to NASA

import httpx
from typing import Optional

NASA_API_BASE = "https://api.nasa.gov/planetary/apod"
API_KEY = "DEMO_KEY"  # Free tier: 30 requests/hour


async def fetch_apod(date: Optional[str] = None) -> dict | None:
    """Fetch Astronomy Picture of the Day from NASA."""
    params = {"api_key": API_KEY}
    if date:
        params["date"] = date

    async with httpx.AsyncClient() as client:
        try:
            response = await client.get(
                NASA_API_BASE, params=params, timeout=30.0
            )
            response.raise_for_status()
            return response.json()
        except httpx.HTTPError:
            return None

async/await

Non-blocking I/O while waiting for NASA

Error Handling

Graceful fallback on network failures

3 First Tool

Your First MCP Tool

@mcp.tool()
async def get_todays_space_photo() -> str:
    """
    Get today's Astronomy Picture of the Day from NASA.

    Returns a beautiful space photo with detailed explanation
    from NASA astronomers.
    """
    data = await fetch_apod()

    if not data:
        return "Unable to fetch today's space photo."

    result = f"""**{data['title']}**
Date: {data['date']}

**What you're seeing:**
{data['explanation']}

**Media:** {data['media_type'].title()}
Link: {data['url']}
"""
    if 'copyright' in data:
        result += f"\nCopyright: {data['copyright']}"

    return result
The docstring is crucial — the AI agent reads it to decide when to call this tool.
Concepts

How MCP Tools Work

When you register a function with @mcp.tool(), here is what happens at runtime:

User asks a question
→
Agent reads docstring
→
Agent calls tool via JSON-RPC
FastMCP runs your function
→
Result returned to agent
→
Agent shows result to user
FastMCP handles all the protocol plumbing — you just write normal Python functions.
4 Photo by Date

Parameters & Validation

@mcp.tool()
async def get_space_photo_by_date(date: str) -> str:
    """
    Get the Astronomy Picture of the Day for a specific date.

    Args:
        date: Date in YYYY-MM-DD format (e.g., "2024-01-15")
              Must be between 1995-06-16 (first APOD) and today.
    """
    try:
        parsed_date = datetime.strptime(date, "%Y-%m-%d")
        first_apod = datetime(1995, 6, 16)
        today = datetime.now()

        if parsed_date < first_apod:
            return f"Date must be after June 16, 1995. You requested: {date}"
        if parsed_date > today:
            return f"Date cannot be in the future."
    except ValueError:
        return f"Invalid date format. Please use YYYY-MM-DD."

    data = await fetch_apod(date)
    if not data:
        return f"Unable to fetch space photo for {date}"

5 Random Photo

Generating Random Parameters

@mcp.tool()
async def get_random_space_photo() -> str:
    """
    Get a random Astronomy Picture of the Day from NASA's archives.

    Selects a random date between June 16, 1995 (first APOD) and today.
    """
    first_apod = datetime(1995, 6, 16)
    today = datetime.now()
    days_diff = (today - first_apod).days

    random_days = random.randint(0, days_diff)
    random_date = first_apod + timedelta(days=random_days)
    random_date_str = random_date.strftime("%Y-%m-%d")

    data = await fetch_apod(random_date_str)
    if not data:
        return "Unable to fetch random space photo. Please try again."
    # ... format and return result
Date arithmetic: Calculate days between first APOD (1995-06-16) and today, pick a random offset, add it back.

 Security

Security Considerations

MCP servers extend what an AI agent can do. That power demands careful security thinking.

General MCP Risks

  • Over-permissive tools — a tool that writes files or runs shell commands can be exploited via prompt injection
  • Data exfiltration — tools that return sensitive data (env vars, DB rows) can leak through the conversation
  • No built-in auth — the MCP spec does not mandate authentication between client and server
  • Secrets in config — API keys hardcoded in server code or JSON configs end up in repos

HTTP / SSE Transport Risks

  • Open network endpoint — SSE transport binds to a port anyone on the network can reach
  • No TLS by default — traffic is plaintext; credentials and responses are sniffable
  • No request auth — any HTTP client can call your tools without a token
  • SSRF / open redirect — tools that accept URLs as parameters can be pointed at internal services

Mitigations

TLS / Reverse Proxy

Put SSE behind HTTPS (nginx, Caddy) so traffic is encrypted in transit.

Auth Tokens

Require a Bearer token or API key header; reject unauthenticated requests.

Network Isolation

Bind to 127.0.0.1, use firewall rules, or run inside a private VPC.

Input Validation

Validate and sanitize every parameter. Allowlist URLs, reject path traversals.

Rate Limiting

Throttle requests per client to prevent abuse and runaway API costs.

Secrets in Env Vars

Load API keys from .env or a secrets manager — never hardcode in source.
